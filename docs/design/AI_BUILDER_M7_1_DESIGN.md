## 一、Builder Agent 定位

Architect Agent (M7) 负责"设计什么"，Builder Agent (M7.1) 负责"怎么生成"。

```
DomainSpec (approved)
    │
    ▼
Snapshot (冻结)
    │
    ▼
BuildPlan (actions)
    │
    ▼
Builder Agent ──→ 代码生成
    │
    ├── models.py
    ├── schemas.py
    ├── repository.py
    ├── service.py
    ├── router.py
    ├── events.py
    ├── seed permissions
    └── alembic migration plan
    │
    ▼
输出到 /generated/{module_name}/
    │
    ▼
Human Review → Merge 到代码库
```

**核心原则**: Builder 只生成，不执行。代码生成到隔离目录，人工审核后合并。

## 二、Builder Agent 架构

### 2.1 输入

```json
{
  "snapshot": {
    "spec_id": "uuid",
    "spec_version": 1,
    "spec_json": { ... }
  },
  "build_plan": {
    "plan_id": "uuid",
    "actions": [
      {"action_type": "create_module", "payload": {...}},
      {"action_type": "create_entity", "payload": {...}},
      {"action_type": "create_capability", "payload": {...}}
    ]
  }
}
```

### 2.2 输出目录结构

```
/generated/{module_name}/
├── __init__.py
├── models.py
├── schemas.py
├── repository.py
├── service.py
├── router.py
├── events.py
├── seed_permissions.py        # Seed 权限扩展
└── migration.sql               # Alembic upgrade 语句
```

### 2.3 执行顺序

```
Action 顺序:
1. create_module       → __init__.py (register 调用)
2. create_permission   → seed_permissions.py
3. create_entity       → models.py
4. extend_entity       → models.py (追加字段到已有实体)
5. create_capability   → router.py (端点) + schemas.py (Request/Response)
6. create_workflow     → models.py (状态机 + WorkflowDefinition)
7. create_ui_page      → ui_schema.json (前端页面描述)
```

## 三、代码生成器

### 3.1 生成器接口

```python
class CodeGenerator(ABC):
    """代码生成器基类"""

    @abstractmethod
    def generate(self, action: dict, context: dict) -> GeneratedFile: ...


@dataclass
class GeneratedFile:
    path: str            # "models.py"
    content: str         # 文件内容
    action: str          # "create" | "append"
```

### 3.2 具体生成器

| 生成器 | 输入 | 输出 |
|--------|------|------|
| `ModuleGenerator` | create_module payload | `__init__.py` |
| `EntityGenerator` | create_entity payload | `models.py` (entity class) |
| `EntityExtendGenerator` | extend_entity payload | `models.py` (追加字段) |
| `SchemaGenerator` | create_capability payload | `schemas.py` (Pydantic Create/Update/Response) |
| `RepositoryGenerator` | create_entity payload | `repository.py` (Repository class) |
| `ServiceGenerator` | create_capability payload | `service.py` (Service class) |
| `RouterGenerator` | create_capability payload | `router.py` (端点) |
| `PermissionGenerator` | create_permission payload | `seed_permissions.py` |
| `WorkflowGenerator` | create_workflow payload | WorkflowDefinition JSON |
| `MigrationGenerator` | create_entity payload | `migration.sql` |

### 3.3 代码模板示例

```python
# EntityGenerator → models.py
def generate_entity(entity: dict) -> str:
    fields = entity.get("fields", [])
    field_lines = []
    for f in fields:
        ftype = f["type"]
        if ftype.startswith("FK→"):
            target = ftype[3:]
            field_lines.append(
                f'    {f["name"]}: Mapped[UUID{" | None" if not f.get("required") else ""}] = '
                f'mapped_column(ForeignKey("{target.lower()}.id", ondelete="RESTRICT"), '
                f'nullable={not f.get("required")})'
            )
        elif ftype.startswith("str("):
            length = ftype[4:-1]
            field_lines.append(
                f'    {f["name"]}: Mapped[str{" | None" if not f.get("required") else ""}] = '
                f'mapped_column(String({length}){", nullable=True" if not f.get("required") else ""})'
            )

    unique = ", unique=True" if f.get("unique") else ""
    return f'''class {entity["name"]}(BaseModel):
    """{entity.get("display_name", entity["name"])}"""

    __tablename__ = "{entity.get("table_name", "")}"

{chr(10).join(field_lines)}
'''
```

## 四、Migration Plan 生成

### 4.1 不自动执行 Alembic

Builder Agent 生成 `migration.sql`（DDL 语句），不执行。

```sql
-- Auto-generated by Builder Agent M7.1
-- Module: quality_inspection
-- Review before executing: uv run alembic upgrade head

CREATE TABLE quality_inspection_orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    inspection_no VARCHAR(30) UNIQUE NOT NULL,
    receipt_id UUID NOT NULL REFERENCES goods_receipts(id),
    inspector_id UUID NOT NULL REFERENCES auth_users(id),
    result VARCHAR(20) NOT NULL DEFAULT 'pending',
    status VARCHAR(20) NOT NULL DEFAULT 'draft',
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE TABLE quality_inspection_lines (...);
```

### 4.2 人工执行流程

```
1. Review generated migration.sql
2. cd backend
3. uv run alembic revision --autogenerate -m "M7.1: quality_inspection"
   (对比 generated SQL 与 autogenerate 的差异)
4. 手动调整
5. uv run alembic upgrade head
```

## 六、安全边界

| 规则 | 实施 |
|------|------|
| **不直接写文件** | 代码输出到内存/返回 JSON，由人工决定是否写入磁盘 |
| **不执行 DDL** | 只生成 migration.sql 文本 |
| **不修改现有模块** | extended_entity 生成 diff 建议，不直接修改 |
| **不覆盖已有文件** | 输出到 /generated/ 隔离目录 |
| **必须基于 Snapshot** | 读取 DomainSpecSnapshot，不读实时 Spec |
| **BuildPlan 必须 approved** | 状态检查 |

| ✅ M7.1 | ❌ 不做 |
